public class NotesEstudiantV03 {
    /**
     * A partir de l'estudiant i la nota, composa un missatge indicant si l'estudiant 
     * aprova o no la UF
     * 
     * @param estudiant: el nom de l'estudiant
     * @param nota: nota de l'estudiant
     * @return missatge composat
     */
    static String composaResultat(String estudiant, int nota) {
        return String.format(
                "L'estudiant %s %s la UF amb un %d.",
                estudiant, 
                nota < 5 ? "suspèn" : "aprova",
                nota
                );
    }

    /**
     * A partir dels noms de estudiants i les seves notes de la UF, 
     * composa un missatge indicant si cada estudiant aprova o no la UF.
     * Nota: es suposa que:
     * - el nombre d'estudiants i de notes és el mateix.
     * - la nota de l'estudiant estudiants[i] correspon a notes[i]
     *
     * @param estudiant: un array de noms d'estudiants
     * @param notes: un array de notes d'estudiants
     * @return missatge composat
     */
    static String composaResultatsDeTresEstudiants(String[] estudiants, int notes[]) {
        String resultat = "";
        for (int i=0; i < estudiants.length; i++) {
            resultat = String.format("%s%n%s", resultat, composaResultat(estudiants[i], notes[i]));
        }
        return resultat;
    }

    public static void main(String[] args) {
        String missatge = composaResultatsDeTresEstudiants(
                new String[] { "Laia Sánchez d'Àrgent",
                               "Cipriano Lavanda Tintín",
                               "Nemesia Gertru i Zimba" },
                new int[] {6, 4, 8});
        System.out.println(missatge);
    }
}

Amb la incorporació d’arrays, el codi millora una mica. Amb tot, aquesta manera de solucionar no acaba de fer el pes.

Un dels problemes que presenta és que hem de suposar que els usuaris de composaResultatsDeTresEstudiants() faran servir la funció passant el mateix nombre d’estudiants que de notes, i que aquests es corresponen per cada posició. Aquest problema el resolem posant la nota als comentaris de la funció i creuant els dits perquè els desenvolupadors el llegeixin.

Hi ha altres problemes de més difícil solució. Per exemple, què passaria si, en comptes de tenir el nom de l’estudiant en una sola variable, el tinguéssim separat en nom de pila i cognoms? Un altre cas, què suposaria si en comptes de la UF actual, ens interessés indicar el nom i el codi de la UF? i nom i el codi del mòdul?

Amb l’aproximació per arrays, aquest tipus de problemes ens porta a passar un array per cada concepte (nomEstudiant, cognomsEstudiant, notaEstudiantEnUF, nomUF, codiUF …) i assegurar-nos sempre que els índexs de cada array coincideixen.

No seria genial disposar de quelcom que ens permetés agrupar tots aquests conceptes en un concepte més gran, com ara Resultat?
