##############################
Estructura d'una aplicació Web
##############################

Un esquema bàsic d'aplicació Web seria:

.. _diagrama_esquema_basic_aplicacio_web:

.. uml::
   :align: center

   @startuml

   skinparam actor {
       backgroundcolor white
   }
   skinparam participant {
       backgroundcolor white
   }

   actor Usuària
   Usuària -> Navegador: //submit//
   Navegador -> Servidor: //Request//\n(GET/POST)
   Servidor -> Servidor: comprovacions
   Servidor -> Navegador: missatge d'error\nex. 404 Not Found
   Servidor -> Aplicació: crida
   Aplicació -> Aplicació: calcula resultat
   Aplicació -> Servidor: resultat\nex. pàgina html
   Servidor -> Navegador: //Response//
   Navegador -> Usuària: resultats "en maco"

   @enduml


Posem noms:

* La usuària ets tu, jo, o qui sigui que faci servir l'aplicació Web

* El navegador és Firefox, Chrome o qualsevol altre

* El servidor és un programa que rep peticions (*Request*) i, a partir
  d'aquestes, decideix cridar a una funció que se n'encarregui de
  generar un resultat amb el que composar una resposta (*Response*)

  En els exercicis que farem a partir d'ara, farem servir el servidor que
  ens ofereix el *microframework* Spark.

* L'aplicació correspon a un dels nostres programes. Podem considerar-la
  com una funció que una entrada i genera una sortida.

  Aquesta és, per tant, la part que ens toca programar.


Exercici 1. A veure si ho he entès bé
=====================================

Respon a les següents preguntes el millor que siguis capaç

1. Com es comunica normalment cadascun dels *actors* del :ref:`diagrama
   <diagrama_esquema_basic_aplicacio_web>`? Amb actors, has d'entendre la
   *Usuària*, el *Navegador* i resta de caixetes.

   Per exemple, la *Usuària* es comunica normalment amb el navegador fent
   clicks al ratolí o bé escrivint amb el teclat.

2. Quines de les comunicacions que mostra el diagrama penses que
   requeriran més temps per arribar de l'origen a la destinació? Amb
   comunicacions has d'entendre les fletxetes.

   Pista: probablement el temps entre que la *Usuària* fa un click al botó
   d'enviar i el moment en que el navegador ho rep, no sigui el més llarg
   en aquesta cadena de comunicacions.

3. Quins dels actors miraran codi HTML?
