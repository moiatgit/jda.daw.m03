############
Requeriments
############

Què presuposo de tu i del teu equip per realitzar aquest curs?

Coneixements previs
===================

Els continguts d'aquest curs estan dissenyats per a poder ser seguits per una persona sense
coneixements previs de programació. S'espera, però, que l'estudiant sigui capaç de manegar
mínimament un ordinador tipus PC, fins el punt d'arrencar aplicacions, navegar per les carpetes i
obrir fitxers, etc.

Tot i que els continguts es realitzaran fent servir un sistema operatiu GNU/Linux, no és pressuposen
coneixements previs d'aquest sistema ni de cap altre.

Requeriments tècnics
====================

Per a poder seguir aquest curs, suposaré que disposes d'un PC amb una
distribució Debian o Ubuntu estàndard, més o menys recent.

El PC no cal que sigui gaire potent, ni tenir molt d'espai de disc lliure,
ni memòria, doncs la major part del temps estarem editant text i executant
codi des de línia de comandes. Això permet que puguis seguir fàcilment el
curs amb una màquina virtual sobre un altre sistema operatiu.

Aquestes distribucions normalment ja porta instal·lat un editor de text
pla com ara ``gedit`` [#editortextpla]_. Si no tens cap altra preferència
et recomano que facis servir ``gedit`` 

.. tip: l'ús d'un editor més *guay* no t'ajudarà a aprendre a programar ni més ràpid ni
    millor.

    El meu consell: **no et compliquis**

En el curs farem servir principalment dos llenguatges de programació:
Python3 i Java

Mentre que molts dels programes que desenvoluparem, podríen funcionar
sense (gaire) problemes amb Python 2.x , es recomana moltíssim fer servir
Python 3.x (per exemple 3.5). Python 3 sol estar pre-instal·lat en les
darreres versions de Debian i Ubuntu.

Addicionalment, algunes activitats poden requerir certes llibreries
que poden necessitar privilegis d'administració. És recomanable instal·lar
els següents paquets:

.. code-block:: sh

    $ sudo apt install python3-pip python3-tk

Gràcies a l'instal·lador de Python ``pip``, serà possible realitzar la
resta de les instal·lacions de mòduls Python sense necessitat de
privilegis d'administració.

Més endavant trobaràs a aquests continguts una guia per instal·lar les
eines de Java sense necessitat de privilegis d'administració.

És perfectament possible realitzar la majoria, si no tots, els exercicis
amb un sistema diferent [#noalumnes]_.  Però no t'ho recomano,
especialment pels exercicis que hagis de lliurar. Si tot i així vols
intentar-ho, per descomptat ho pots fer **al teu risc**. Et quedarà com a
exercici trobar com realitzar les diferents tasques que se't requeriran
amb el sistema operatiu escollit, així com resoldre les possibles
incompatibilitats (especialment amb la codificació de caràcters), etc.

----

.. [#editortextpla] Com que hi ha molts editors de text pla que siguin adequats, potser és
    preferible deixar clar què **no** és un editor de text *pla*. Per exemple no seria adequat que
    programessis amb ``LibreOffice Writer`` o ``MS Word``. Si tens dubtes de si el teu editor és o no
    de text pla, agafa un fitxer editat per aquest i en una consola fes:

    .. code-block:: sh

        $ cat nomfitxereditat

    Si et mostra el contingut tal i com l'havies escrit al teu editor,
    pots fer servir aquest últim per editar els teus programes amb
    tranquil·litat.

.. [#noalumnes] Evidentment, si estàs fent servir aquests apunts per lliure i no m'has de lliurar
    cap exercici, fes servir el SO, l'editor i, fins i tot, el llenguatge de programació que et
    vingui de gust amb la confiança de que serà totalment possible!
